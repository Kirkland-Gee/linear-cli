## Suggested conventions
- Use repo docs for implementation detail.
- Use Linear for active execution, prioritization, and progress tracking.
- Put one clear outcome per issue.
- Add `agent:*` labels based on the default worker role.
- UI changes should link screenshots in the issue or PR.
- Keep backlog curated; do not turn every thought into a ticket.
